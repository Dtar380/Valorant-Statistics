API

This API has been made public so everyone can access the collected data. The API is free to use and does not require keys.

The API is the same as the one used on this website, so you can retrieve the data in the same way as it's presented on the website.

STATICS

The "statics" route is used to retrieve static data from the API. These are non-relational data, meaning data from specific columns.

Headers

Content-Type string

Route Parameters

column string
The "column" parameter corresponds to one of the 5 existing columns. These are:
  • AGE
  • OCCUPATION
  • GENDER
  • PEAK
  • ROW VALUE
return_type string
The "return_type" parameter corresponds to how the data will be returned. These are:
  • table: returns the entire table
  • graph: returns the table prepared for graphing
  • mode: returns the most frequent value

Request Examples

QUERY

The "query" route is used to make queries to the API. Queries always return relational data due to how they work.

Headers

Content-Type string

Route Parameters

query_type string
The "query_type" parameter corresponds to the type of query being made. These are:
  • Combination: Combine a set of data
  • Comparison: Compare two columns based on a data point
return_type string
The "return_type" parameter corresponds to how the data will be returned. These are:
  • table: returns the entire table
  • graph: returns the table prepared for graphing

Payload Parameters

column string
The "column" value in the payload corresponds to the name of the column you want to query. These are:
  • AGE
  • OCCUPATION
  • GENDER
  • PEAK
value string
The "value" in the payload corresponds to the value you want to query.
